Conclusion

Based on explicit and implicit sections, Webkit browsers will style h1 Elements properly. For example, an h1 in a section in the body semantically acts as a level-2 heading. Therefore the Webkit browser will style it as if it were an h2.

However, all other headings, h2–h6, have constant styles. That is, an h2, no matter want it means (semantically, based on explicit/implicit sectioning), will always appear as an h2.
