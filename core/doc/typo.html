Paragraphs

A p Element represents a generic paragraph.

This is a generic blockquote. Use this for material that is more than 40 words and/or contains paragraph breaks.

Nest p Elements inside the block quotation that correspond directly to the paragraphs of the quoted material. If there are no paragraphs in the quoted material, do not use p Elements—just simply contain the text inside the blockquote.

To cite the source of the quoted material in this block quote, include a footer > cite Element.

Tables

tl;dr: Add class .Table to a table Element to make it appear as an encapsulated unit (like a paragraph). In the .Table, use .Rowgroup on thead, tfoot, and tbody Elements to add the borders above and below. Use subclasses .Head, .Foot, and .Body for specific display properties. Use .Row on tr Elements to add odd and even (or head) coloring. Use .Cell on all th and td Elements to add padding and min-height. Use subclass .Hc on th for a heading look (font-family and bold).

By defualt, tables are horizontally oriented. Horizontal tables are best for displaying information that correlates across rows. An example of this might be a list of student information including names, ages, genders, and ID numbers. Add the Interface .u-w_1o1 to stretch the table to the full page width.

Below is a .Table.Vert. Vertical tables are best for displaying information that correlates down columns. An example of a vertically-oriented table might be a spreadsheet displaying the position, velocity, and acceleration values of a falling object across time.
