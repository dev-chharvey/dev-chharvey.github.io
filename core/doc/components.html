The .c-Subhead Component

Put a .c-Subhead sibling adjacent to a heading to create a subheading for a page. This can be thought of as a tagline or subtitle for the top-level heading.

Notice the differences between the title and subtitle. The title is in a sans-serif, monoweight type, in bold and small caps, which provides a very rigid and blocky look. The dark blue color itself contrasts the font and gives the title a cool and calming appearance. In great contrast, the subtitle is in a script italic type with a dramatic thick/thin transition, which looks fragile and dainty. The warmer color, however, gives it a strong pop.

The .c-FolioTitle Component

On special pages, the top-level header may be a .c-FolioTitle. To add a little spice, use span.c-FolioTitle__Cap on capital letters. The size will make them stand out.

The .c-Pullquote Component

A .c-Pullquote tends to have a stronger effect than generic blockquote Elements. Use it when the original source is less than 40 words and contains no paragraph breaks. Do not include child p Elements. To cite the source, add a child .c-Pullquote__Src Component.

The .c-Table* Family of Components

Use the following Components for extra styles on table and dl Elements.

.c-TableBordered
border around the entire table
.c-TableBorderedRows
borders in between rows
.c-TableBorderedCols
borders in between cells in rows
.c-TableShaded
background shading to the entire table
.c-TableShadedRows
background shading to alternating rows
.c-TableShadedCols
background shading to alternating cells in rows
.c-TableHover
background shading to a row on hover

Notice: These Components are all independent of each other, that is, the use of one does not require another. For example, a table may be classed .c-TableBorderedRows but may not necessarily be classed .c-TableBordered.

The .c-Document Component

A .c-Document may take the form of a body > main or body >> article. The said document should have a status and be marked up with metadata. Read my technical report on document status for more information.

Any Element inside a .c-Document may be marked with classes indicating their status. Notice we are separating semantics with presentation: the [data-status] attribute is unrelated to an Element’s class.

A Document is normative by default, but may also have informative (non-essential) Elements.

The .c-Alert Component

The Alert Component builds off the Box Object, so you must include o-Box in the class attribute. The Alerts shown below all implement the subclass .o-Box--island, but it may be replaced with .o-Box--dialog or .o-Box--hero. Subclasses of the Alert Component determine specific types of Alerts. The Alert may contain an optional .c-Alert__Title Subcomponent at the beginning.

The .c-Label Component

The Label Component builds off the Textbox Object, so include the o-Textbox class. Subclasses add skins.
